1. Refresco automático de ListaProductosPage al modificar un producto

   Problema reportado:
   - Al actualizar un producto desde ModificarProductoPage, la lista de
	 productos en ListaProductosPage no reflejaba los cambios. El usuario
	 debía navegar manualmente para ver los datos actualizados.

   Causa raíz:
   - El código usaba Application.Current.MainWindow para obtener la
	 MainWindow, pero esta propiedad apuntaba al LoginView (la primera
	 ventana mostrada en App.OnStartup), no a MainWindow. Por lo tanto,
	 el cast "is MainWindow" siempre daba false.

   Solución aplicada:
   - Se cambió Application.Current.MainWindow por Window.GetWindow(this),
	 que obtiene la ventana real donde está alojada la Page.
   - Se hizo público el método CargarProductos() en ListaProductosPage.
   - Se agregó el método RefrescarListaProductos() en MainWindow como
	 puente para invocar la recarga.

   Archivos modificados:
   - Invetario/Views/ModificarProductoPage.xaml.cs
   - Invetario/Views/ListaProductosPage.xaml.cs
   - Invetario/Views/MainWindow.xaml.cs

   Estado: ✅ Aplicado y funcional

4. Filtrado por categoría en ListaProductosPage

   Descripción:
   - El ComboBox de categorías ahora filtra la lista de productos al
	 seleccionar una categoría.
   - Se agregó la opción "Todas" (Id=0) como primer elemento del combo,
	 seleccionada por defecto.
   - El filtro de texto y el de categoría funcionan combinados: se aplican
	 ambos simultáneamente.
   - Se mejoró la búsqueda por código de barras para usar Contains en
	 vez de igualdad exacta (permite buscar mientras se escribe).

   Problema adicional corregido:
   - El filtrado no funcionaba inicialmente porque CargarProductos() se
	 llamaba antes de CargarCombo(), y el evento SelectionChanged se
	 disparaba prematuramente durante la inicialización.
   - Se reordenó la inicialización (primero combo, luego productos).
   - Se agregó un flag _inicializado para evitar ejecutar filtros antes
	 de que todo esté listo.
   - Al recargar productos, se re-aplican los filtros activos.

   Archivos modificados:
   - Invetario/Views/ListaProductosPage.xaml
   - Invetario/Views/ListaProductosPage.xaml.cs

   Estado: ✅ Aplicado y funcional

7. Ajuste responsivo de todas las vistas

   Problema reportado:
   - Varias vistas no mostraban todo el contenido a menos que el usuario
	 agrandara manualmente la ventana. Los DataGrids tenían alturas fijas
	 y los contenedores StackPanel no se estiraban.

   Solución aplicada:

   a) MainWindow.xaml:
	  - Se agregó WindowState="Maximized" para que se abra a pantalla
		completa.

   b) ListaProductosPage.xaml:
	  - Se reemplazó StackPanel por DockPanel como contenedor raíz.
	  - Se eliminó Height="450" fijo del DataGrid; ahora se estira para
		llenar el espacio disponible.
	  - Paginación anclada al fondo (DockPanel.Dock="Bottom").

   c) VentasPage.xaml:
	  - Se reemplazó StackPanel por DockPanel.
	  - Se eliminó Height="300" fijo del DataGrid del carrito.
	  - Total anclado al fondo.

   d) AdminUsuariosPage.xaml:
	  - Se reemplazó StackPanel por DockPanel.
	  - Se eliminó Height="350" fijo del GroupBox.

   e) ModificarProductoPage.xaml:
	  - Se envolvió el contenido en un ScrollViewer para que el
		formulario sea navegable si la ventana es pequeña.

   f) NuevoProductoPage.xaml:
	  - Mismo tratamiento con ScrollViewer.

   g) NuevaCategoriaPage.xaml:
	  - Se eliminaron Height="450" Width="800" fijos que limitaban el
		tamaño de la página.

   Archivos modificados:
   - Invetario/Views/MainWindow.xaml
   - Invetario/Views/ListaProductosPage.xaml
   - Invetario/Views/VentasPage.xaml
   - Invetario/Views/AdminUsuariosPage.xaml
   - Invetario/Views/ModificarProductoPage.xaml
   - Invetario/Views/NuevoProductoPage.xaml
   - Invetario/Views/NuevaCategoriaPage.xaml

   Estado: ✅ Aplicado y funcional
